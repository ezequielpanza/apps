Contextum Privacy Policy

Effective date: July 10, 2026

What Contextum does

Contextum is an opt-in context bridge. When a user explicitly enables cloud sync, Contextum can upload a current context snapshot so the user can temporarily access it from compatible services, including a custom GPT Action.

Data processed

A context snapshot may include precise latitude and longitude, GPS accuracy, speed, heading, altitude, timestamps, an optional active note entered by the user, and limited runtime state such as whether the Contextum page is visible, focused, or in Companion Mode.

When data leaves the device

Cloud sync is disabled by default and requires explicit activation by the user. When disabled, Contextum does not upload the context snapshot through this cloud bridge.

Temporary access codes

Users may generate an 8-character temporary code. A valid code grants read-only access to the associated context snapshot. Codes expire after approximately 10 minutes. Anyone who obtains a still-valid code may be able to read that snapshot, so users should treat codes as temporary secrets.

Retention

Uploaded context snapshots are stored in Contextum's Cloudflare KV-backed bridge with a time-to-live of up to 30 days from the latest stored snapshot. Temporary code mappings expire after approximately 10 minutes. Contextum may revise retention as the product evolves; material changes will be reflected in this policy.

Purpose and sharing

Context data is processed to provide the user-requested Contextum service, including retrieval through a temporary code or a configured GPT Action. Contextum does not sell personal data. When a user asks ChatGPT or another external service to retrieve Contextum data, that service may receive the temporary code and the returned context needed to fulfill the request. The external service's own privacy terms also apply.

Security

Contextum uses HTTPS endpoints, no-store response headers for context responses, hashed device-key addressing in the cloud bridge, and short-lived temporary access codes. No system can guarantee absolute security.

User choices

Users can keep cloud sync disabled, avoid generating temporary codes, remove optional active notes, or stop using Contextum. Future versions may add account-based deletion and OAuth controls.

Changes

This policy may be updated as Contextum adds authentication, memory, additional sensors, or other capabilities. The effective date will be revised when material changes are made.

Service

Contextum is currently an experimental project. Privacy questions should be raised through the support or project contact channel provided with the Contextum distribution.
